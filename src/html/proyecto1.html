Objetivo

El objetivo de este proyecto fue diseñar un sistema para gestionar las incidencias relacionadas con equipos informáticos o audiovisuales dentro de una organización. Este sistema debía permitir la recopilación de datos mediante un formulario, la transformación de esos datos en un formato estándar (CSV a XML), y luego procesar y almacenar la información en diferentes formatos, como JSON, para su posterior análisis. Además, se debía asegurar que el sistema cumpliese con los estándares de sostenibilidad, considerando la reutilización de los datos y su almacenamiento en formatos accesibles a largo plazo.

Resolución

El primer paso fue la creación de un formulario en línea para recabar las incidencias de los usuarios. Utilicé Google Forms para diseñar el formulario, permitiendo a los empleados de la organización reportar problemas con los equipos informáticos y audiovisuales. Los usuarios introdujeron información como el tipo de incidencia, la fecha y la descripción del problema, entre otros detalles.

Tras recoger las respuestas en el formulario, exporté los datos a un archivo CSV desde Google Sheets. Posteriormente, utilicé una herramienta de programación en Python para realizar una conversión de los datos de CSV a XML. Este paso fue clave, ya que XML es un formato más adecuado para el intercambio de datos estructurados y tiene mayor compatibilidad con sistemas automatizados.

Escribí un programa en Python que leyó el archivo XML generado en el paso anterior. El objetivo era procesar las incidencias y mostrar la información de manera más legible y útil para un administrador de sistemas. El código permitía mostrar las incidencias por categoría, destacando la información más relevante, como el tipo de problema y la fecha de reporte.

Posteriormente, añadí una funcionalidad al programa en Python para almacenar los datos procesados en un archivo JSON. Este paso fue crucial para asegurar la portabilidad de los datos y su almacenamiento en un formato fácilmente accesible en el futuro. El archivo JSON generado contenía toda la información procesada, estructurada en un formato que podía ser utilizado por otras aplicaciones o plataformas.

Conclusión

Este proyecto no solo cumplió con los objetivos técnicos de gestionar y procesar las incidencias informáticas, sino que también permitió aplicar principios de sostenibilidad al seleccionar formatos de archivo accesibles a largo plazo (CSV, XML, JSON). Al integrar las mejores prácticas de recolección y transformación de datos, así como la importancia de los ODS, pude contribuir a una infraestructura más eficiente y sostenible dentro de la organización. El uso de formatos abiertos y no propietarios, como XML y JSON, garantiza que los datos recopilados sean accesibles y reutilizables en el futuro, contribuyendo al desarrollo de un entorno profesional más sostenible.
